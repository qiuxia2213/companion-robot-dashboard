# 作业二反思：从"看着它"到"看见它"

做伴侣机器人研究的这段时间，我每天最固定的动作就是看日报——一份每天自动抓取的、关于这个议题的新闻、论文与社群讨论的文字汇总。它信息密度很高，分类也清晰，但一旦累积起来，我开始感到它的"无重量感"。每条新闻、每篇论文、每段帖子都以纯文本的方式平摊在我眼前，我能读懂每一条，却看不见它们之间的关系。

这种困境是做理论研究的人很熟悉的——文字本身是线性的，但我们要研究的世界是网状的。哪些媒体在用什么样的话语谈论伴侣机器人？批判性话语和宣传话语在时间上的此消彼长是怎样的？中国和海外的报道在框架上呈现了哪些差异？学术论文和大众媒体在术语上是否互相对话？这些问题，理论上都"在数据里"，但藏在一堆 Markdown 文件下面，我每天都"看着它"却"看不见它"。

想到做可视化，某种意义上不是技术决定，而是一种认知层面的自救。我希望能跳出"今天又看了几篇"的字面阅读，转而看见信源结构、立场分布、关键词频次、时间趋势这些只有从高处才能看见的形状。文字让我"知道事实"，图表让我"理解模式"——后者才是研究者真正需要的视角。

但开始动手以后，过程比我想象的漫长得多。有想法和能实现之间隔着一道沟，而填这道沟需要不停地报 bug、问 AI、等回答、再调、再问。我能写出研究计划、能读懂福柯，但当 Streamlit 抛出一个 TypeError 时，我需要把这个错误和"我到底想看什么"一遍遍重新对齐。这种"工程问题"和"研究问题"不断切换的疲惫，是我以前没有体验过的。
